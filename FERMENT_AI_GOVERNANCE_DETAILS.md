| 5,001人〜 | 全顧客超え、配信ミスは即ブランド毀損 |

## A-3. Kill Switch（緊急停止）

### 配置場所
ダッシュボード上部に常設の赤いボタン「🛑 全自動停止」

### 停止対象
- 全 cron jobs（一時停止フラグ）
- AI エージェント呼び出し
- 配信キュー処理

### 解除条件
- 別の人間（オーナー権限）の承認
- 解除理由のテキスト入力（監査ログに記録）

### 自動発動条件
- 5分以内に同一エラー10回以上
- バウンス率 20% 超過
- 配信失敗率 30% 超過
- AI API コスト 1日 ¥3,000 超

## B-2. 帰属売上目標の根拠

### Phase B 目標：月 ¥200,000

#### 計算式
```
帰属売上 = 配信通数 × クリック率 × CV率 × 平均購入額
```

#### 試算
| 項目 | 値 |
|------|---|
| 月間配信通数 | 15,000通（月3-4キャンペーン × 平均4,000通）|
| クリック率 | 3% |
| クリック後 CV率 | 5% |
| 平均購入額 | ¥6,000 |
| **帰属売上** | **¥135,000/月** |

→ Phase B 目標 ¥200,000 は **やや野心的**だが、AI 最適化で達成可能。

### Phase C 目標：月 ¥500,000

#### 達成戦略
- 配信通数 30,000/月（細分化セグメント・行動トリガー）
- クリック率 4%（パーソナライズ）
- CV率 6%（個人別レコメンド）
- 平均購入額 ¥6,500

#### 試算
```
30,000 × 4% × 6% × ¥6,500 = ¥468,000
```
→ Phase C 目標 ¥500,000 は **達成可能**。

### Phase D 目標：月 ¥1,000,000

#### 達成戦略
- 行動トリガー配信を全自動化
- 個別最適化された配信時刻
- AI が学習し続けたテンプレ

#### 試算
```
50,000 × 5% × 8% × ¥7,000 = ¥1,400,000
```
→ Phase D 目標 ¥1,000,000 は **十分達成可能**。

# C. 学習データ・プライバシー設計

## C-1. データ階層化

| 階層 | 例 | AI 利用可 | 保存期間 |
|------|---|---------|---------|
| **公開データ** | 業界ベンチマーク、商品マスタ | ✅ | 永続 |
| **集計データ** | セグメント別開封率 | ✅ | 5年 |
| **匿名化データ** | ハッシュ化 customer_id | ✅ | 2年 |
| **PII（個人特定情報）** | email, name, phone | ⚠️ 限定 | 不要時削除 |

## C-2. AI への個人情報渡し方

### 戦略エージェントへ渡すもの（OK）
- セグメント単位の集計
- 顧客プロファイル（匿名化 ID）
- 行動パターン（タグ・購入頻度）

### 戦略エージェントへ渡さないもの（NG）
- 具体的な顧客名
- メールアドレス
- 電話番号
- 個別の購入商品名（除：レコメンド時の本人向け）

### 個別配信時のみ渡すもの
- 配信先の顧客プロファイル（パーソナライズ用）
- ただし配信完了後に AI コンテキストから削除

## C-3. データ最小化原則

### 学習データから除外
- 過去の配信本文の生データ → 集計指標のみ保存
- email アドレス → ハッシュ化保存
- 失敗配信の詳細エラー → エラータイプのみ

## C-4. ユーザーの権利

### GDPR 準拠（既存実装）
- データ削除リクエスト → 自動処理
- データエクスポート → CSV提供（既存）
- 利用停止リクエスト → 配信停止 + 学習データから除外

### 透明性
- 「あなたのデータがどう使われているか」ページを将来追加
- メール内に「このメールは AI が生成した内容を含みます」表示（任意）

# D. ブランド保護・「オリゼらしさ」の定義と維持

## D-1. ブランドガイドラインの言語化

### オリゼらしさの 5 要素

| 要素 | 定義 | 例 |
|------|------|-----|
| **誠実さ** | 過剰な煽りなし、事実ベース | ✅「米麹発酵で〇〇」 ❌「奇跡の食品！」 |
| **温かみ** | 人間味のある言葉遣い | ✅「{{name}}さん、今日も元気にお過ごしですか？」 |
| **専門性** | 米麹・発酵への深い知識 | ✅「アルファ化された米麹が...」 |
| **健康志向** | 食と健康の繋がり強調 | ✅「腸内環境を整える」 |
| **日本の伝統** | 日本食文化の継承 | ✅「古来から愛されてきた...」 |

### NG 表現リスト
| カテゴリ | NG ワード |
|---------|----------|
| 誇大広告 | 「奇跡」「魔法」「100%」「絶対」 |
| 健康効能（薬機法） | 「治る」「病気が」「効く」 |
| 過剰な感嘆符 | 「!!!」「？？？」 |
| カタカナ多用 | 「ヘルシー」「ナチュラル」過剰 |
| 焦り煽り | 「今すぐ」「急いで」「ラスト」 |

### NG 表現自動検出
```typescript
const NG_PATTERNS = [
  /奇跡|魔法|100%確実/,
  /(治|効果)が?あ?る/,  // 薬機法
  /[!！]{3,}/,
  /今すぐ|急げ|ラスト/,
];
```
AI 生成テンプレで検出 → 自動再生成 or 人レビュー。

# F. 緊急時対応（AI 暴走・誤配信）

## F-1. シナリオ別対応プレイブック

### シナリオ 1：AI が誤った内容で大量配信した

#### 即時対応（5分以内）
1. **Kill Switch 押下** → 残りキューを停止
2. **送信済み件数の特定** → wrangler tail でログ確認
3. **影響範囲の評価** → 開封率・反応速度

#### 24時間以内
4. **謝罪メール送信** → 影響受けた顧客全員に
5. **社内共有** → Slack で経営層に報告
6. **AI 設定の修正** → プロンプト・ガードレール強化
7. **監査ログ提出** → 何が・なぜ起きたか文書化

#### 中長期
8. **再発防止策実装** → ガード追加、人レビュー強化
9. **シミュレーションテスト** → 似たケースで再発しないか検証

#### 謝罪メールテンプレ（事前準備）
```
件名: 【お詫び】先ほどのメール配信について

{{name}}様

本日、配信ミスにより{{name}}様にご迷惑をおかけしましたこと、
心よりお詫び申し上げます。

[原因の簡潔な説明]

今後はこのようなことがないよう、配信前のチェック体制を
強化してまいります。

引き続きオリゼをよろしくお願いいたします。

株式会社オリゼ 一同
```

9. AI プロンプトに NG パターン追加
10. 同様のテンプレ全体見直し

### シナリオ 3：AI が暴走して大量に API 叩く

#### 自動検知（既存）
