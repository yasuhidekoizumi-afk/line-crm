## 2. 設計方針

### 2-1. なぜ公式で作るか（中断していたQ1の確定）

| 観点 | 判断 |
|---|---|
| 役割分担 | ウェルカムはマス自動配信 → **公式が正しい置き場所**（Harnessは頭脳に専念） |
| トリガーの信頼性 | 公式の「友だち追加」トリガーは確実に発火。Harnessはデータ破損中（友だち実数の半分しか把握できず）でトリガー発火が信頼できない |
| 原稿資産 | 既に公式側に `13913 伴走` がある。ゼロから作るより速く・質も担保できる |
| 通数 | 公式・Harnessどちらで送っても通数枠を消費（逃げられない）。ならば運用がシンプルな公式に寄せる |

→ **公式のステップ配信で、`13913 伴走` を土台に再構築する。**

### 2-2. 設計原則

1. **新規作成 or 既存改修か**: `13913` を直接いじると過去の配信実績ログが混ざるリスクがある。
   **新規ステップ配信を1本作り、13913の構成・原稿を下敷きにする**のを推奨（小泉さん判断）。
2. **開始条件**: 「友だち追加：すべて」（経路で分けない。まずは全員一律のシンプルな1本）。
3. **ステップ数**: 初版は **4ステップ**に圧縮（13913の5通は冗長気味。離脱を抑える）。
4. **CTA**: 各ステップに「次の一歩」を1つだけ置く（商品ページ / クーポン / アンケート）。
5. **計測**: トラッキングリンクを各CTAに付け、どのステップが効くか後で測れるようにする。

## 3. ウェルカムフロー初版（設計案）

> 13913「伴走」をベースに、現行の主力商品（KOJIPOP・麹甘味料SOY/SWEET/SALT・甘酒・グラノーラ）に合わせて再構成。
> **§1-5のあいさつメッセージと補完関係になるよう設計（重複を排除）。**
> ※各原稿は実装時に最新の商品URLに差し替える。以下は**目的と骨子**。

**設計上の大前提（あいさつメッセージとの役割分担）:**
- あいさつ（0通目）が既に「お礼＋送料無料クーポン`welcome`＋全商品カルーセル」を担っている。
- → ウェルカムフローは **お礼とクーポンと"商品の幅見せ"は繰り返さない**。
- → 役割を「**想いで惹きつけ → 1つの使い方で深く刺し → 関係を続ける**」に絞る。あいさつ=広く / ウェルカム=深く。

```
（0通目）あいさつメッセージ ＝ お礼＋送料無料クーポンwelcome＋商品カルーセル（実装済・稼働中）
   │
開始条件：友だち追加（すべて）
   │
   ▼ 2日後（10:00-20:00） ※あいさつ翌日以降に置き、被りを避ける
【STEP1｜ブランドの想いで惹きつける】
  - ORYZAEの世界観（13913 STEP1「想い」を踏襲）。お礼は繰り返さず想いに特化
  - CTA: ブランドストーリー記事 or 人気No.1商品
   │
   ▼ 4日後（12:00-13:00）
【STEP2｜1つの使い方で深く刺す】
  - 主力1商品の一番カンタンな使い方（13913 STEP2のドレッシング訴求が秀逸＝流用）＋写真
  - あいさつが"幅"なので、ここは"1点突破"で「これなら使える」を作る
  - CTA: 該当商品の購入ページ（あいさつのwelcomeクーポンがまだ使える旨を添える）
   │
   ▼ 7日後（12:00-13:00）
【STEP3｜レシピで生活に定着させる】
  - 13913 STEP3レシピ集（きのこソテー/じゃがいも塩ソテー等）が流用可。「使い続けられる」を訴求
  - CTA: レシピで使った商品 or レシピまとめページ
   │
   ▼ 30日後（21:00-22:00）
【STEP4｜関係維持＋声を集める】
  - 購入者へは活用・レビュー依頼、未購入者へは「welcomeクーポンまだ使えます」の最終リマインド
  - CTA: アンケート（13913 STEP5）or レビュー依頼
```

> クーポンは原則あいさつの`welcome`（送料無料・無期限）を使い回す。
> 別途「初回限定」を出すかは小泉さん判断（§5-B）。出さなくてもフローは成立する。

## 4. 実装手順（小泉さんの確認後）

1. 公式管理画面 → ステップ配信 → 「作成」で**新規1本**を作る（タイトル例: `2026 ウェルカム v1`）。
2. 開始条件「友だち追加：すべて」、タイムゾーン Asia/Tokyo。
3. STEP1〜4を §3 の骨子で作成。原稿は 13913 を開いて該当文面をコピー → 現行商品に手直し。
4. 各CTAにトラッキングリンクを設定（計測のため）。
5. **テスト配信**で自分のLINEに送り、表示・リンク・絵文字を確認。
6. 通数上限の余裕を確認（§5）してから「利用開始」。
7. **旧ステップ配信（13913等7本）は停止のまま** or アーカイブ。二重発火を防ぐ。
