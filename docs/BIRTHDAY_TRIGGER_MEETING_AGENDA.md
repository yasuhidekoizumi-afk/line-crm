# 【会議議題】誕生日トリガー配信 — 特典の方向性を決める

> 作成: 2026-06-02 / 起案: 小泉 + Claude / 対象: LINE Harness CRM（既存顧客向け誕生日施策）
> **この会議で決めたいこと: 誕生日特典を「A送料無料 / B金額OFF / Cおまけ」のどれにするか**

---

## 1. 何をやろうとしているか（30秒サマリ）

既存顧客の**誕生日当日に、自動でLINE/メール配信＋特典**を届ける施策。
- 対象: 誕生日を登録済みの既存顧客（**1万人以上**）
- 目的: リピート促進・関係維持・ファン化（※新規獲得ではない＝新規に食われるリスクなし）
- タイミング: 誕生日当日（毎日の自動処理で抽出）
- **箱はすでに揃っている**（誕生日データ・クーポン発行機能・配信機能・日次バッチ）。特典を決めれば実装できる。

---

## 2. 前提となる事実（リサーチ＆実データ済み）

| 項目 | 事実 |
|---|---|
| 既存客の客単価(AOV) | **約3,104円**（直近90日）|
| 既存客がよく買う商品 | 麹マヨ1,378円・CHOCOソース1,038円など**低単価の単品買いが主力** |
| 送料無料ライン | **5,000円以上**（国内・通常便）|
| 送料 | 280円（〜0.7kg）/ 480円（以上）|
| おまけ候補の最安商品 | ソース類750円（醤油/塩/甘麹・CHOCOソース）|
| 競合の誕生日特典相場 | 定額500円OFF（GU・ベルメゾン）/ 10〜20%OFF（GAP・クリスピー）/ ポイント500〜1000pt（コスメ）/ 無料プレゼント（サーティワン・ロクシタン）|

### ⚠️ 重要な制約：Shopifyの送料無料は「割引後金額」で判定される（公式仕様・変更不可）
- 例: 5,400円のカートに500円OFF → 4,900円 → **送料480円が復活**してしまう
- 「誕生日プレゼントのつもりが送料を取られてガッカリ」が起きる
- Shopifyは「これは仕様、直さない」と公式明言。アプリ導入以外に標準回避策なし
- → **割引額のある特典（B）は、この問題への対処が必須**

---

## 3. 選択肢（比較表）

> ※ おまけプレゼント案（C）は物流同梱オペが重いため **今回は見送り**。デジタル完結の2案に絞る。

| | **A. 送料無料クーポン** | **B. 500円OFFクーポン** |
|---|---|---|
| 顧客の感じる価値 | 280〜480円 | 500円（明快）|
| **実原価** | 280〜480円 | 最大500円 |
| 送料復活問題 | ✅ 起きない | ❌ 起きる→利用条件6,000円以上が必要 |
| 特別感・ギフト感 | △ 機能的 | △ 事務的 |
| 低単価リピート層に効くか | ◎ | ✗（6,000円ハードル）|
| 自動化のしやすさ | ◎ デジタル完結 | ◎ デジタル完結 |
| 在庫・物流負荷 | なし | なし |

---

## 4. 論点（会議で意見が割れそうな点）

1. **「気軽な再訪」か「まとめ買い」か** — Aは低単価の単品買いを後押し（ハードルを下げる）。Bは利用条件6,000円以上が要るのでまとめ買い誘導になるが、主力の低単価リピート層が使えなくなる。
2. **原価をどこまで許容するか** — 1万人×特典の年間コスト感。AもBも上限は同程度（〜500円弱）だが、Aは送料実費で固定、Bは利用された分だけ。
3. **割引率の議論をするなら** — Bで%OFF（15%等）にすると送料問題＋原価変動が両方乗る。定額が無難。
4. **将来のアップグレード** — おまけプレゼント(C)は物流連携が整えば再検討の余地あり。今回はA/Bで立ち上げる前提。
