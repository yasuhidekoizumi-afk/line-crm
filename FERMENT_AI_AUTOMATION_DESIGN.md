# FERMENT AI 全自動 CRM 設計書

**作成日**: 2026-04-25  
**対象**: 株式会社オリゼ（ORYZAE Inc.）  
**現状**: Phase A（手動）  
**目標**: Phase D（完全自動 CRM）

---

## 📐 0. 設計原則

### 1. **段階的移行**
完全自動を目指すが、一気に切り替えない。半自動 → 監督付き自動 → 完全自動の3段階。

### 2. **人間の介入ポイントを明示**
最終判断は常に人間に委ねる箇所を設計時に決める（暴走リスク回避）。

### 3. **学習ループ必須**
全アクションの結果を記録 → AI が学習 → 次の判断改善。

### 4. **コスト制御**
AI API 呼び出しは月額予算上限を設定。超過時は自動停止。

### 5. **可逆性**
全自動アクションには「取り消し」機能を付ける（メール除く・送信後は不可なので承認制）。

## 🤖 2. AI エージェント詳細設計

### 2-1. 戦略エージェント（Strategy Agent）

#### 役割
日次で全データを分析し、「今日やるべきマーケアクション」3つを提案。

#### 入力データ
```json
{
  "today": "2026-04-25",
  "customers": {
    "total": 12435,
    "subscribed_email": 5523,
    "high_intent_30d": 234,
    "churning": 1820
  },
  "recent_campaigns": [
    {"name": "月次ニュースレター", "open_rate": 28.4, "ctr": 3.2, "revenue": 142000}
  ],
  "events_last_7d": ["カゴ落ち急増 +45%", "新商品リリース"],
  "calendar": ["連休中", "母の日近い"]
}
```

#### モデル
Gemini 3 Flash Preview（戦略立案・推論。コスト最適化のため Gemini 統一）

#### システムプロンプト
```
あなたはオリゼ（米麹発酵 EC）のマーケ戦略責任者です。
データを見て、今日やるべきアクション TOP 3 を JSON で返してください。
- 売上インパクト・実行容易性で優先順位
- 各アクションは「対象セグメント・配信内容・想定効果」を明記
- 既存のフロー・テンプレで実行可能なもの優先
```

#### 出力
```json
{
  "actions": [
    {
      "rank": 1,
      "title": "母の日キャンペーンメール",
      "segment": "全メール購読者",
      "template": "月次ニュースレター JP",
      "customizations": "本文に母の日特集追加",
      "expected_impact": "売上 +¥80,000",
      "execute_url": "/email/campaigns/new?segment=...&template=...",
      "reasoning": "母の日まで2週間、過去の母の日メール開封率35%"
    }
  ],
  "warnings": ["カゴ落ち急増中、休眠復帰メール優先度上昇"]
}
```

#### 実行頻度
- 毎日 09:00 JST（cron）
- ダッシュボード表示時にキャッシュから取得

#### コスト
1日1回 × 月30回 × Gemini 3 Flash Preview ~$0.01/回 = **月¥45 程度**

### 2-2. 文面生成エージェント（Content Agent）

#### 役割
シーンに応じてメール HTML を完全生成（既存の AI 本文生成を拡張）。

#### 入力
- 配信目的（例：「母の日特集」）
- 対象セグメント特性
- ブランドキット
- 過去の高開封率テンプレ参考

#### モデル
- 件名: Gemini 3 Flash Preview（既存）
- 本文: Gemini 3 Flash Preview（Claude → 変更、API統一のため）

#### 改善点（現状から）
1. **過去成績学習**：subject_line_history を参考に
2. **ブランドキット自動適用**：色・フォント・ロゴを統一
3. **マルチバリアント生成**：A/B/C 自動生成

---

### 2-3. セグメント自動発見エージェント（Segment Discovery Agent）

#### 役割
顧客クラスタリングで「気付かなかった有望セグメント」を発見。

#### アルゴリズム
- 顧客特徴量：LTV、購入頻度、注文間隔、最終購入経過、開封率、地域、タグ
- K-means クラスタリング（k=5-10）
- 各クラスタの特徴を Claude が言語化

#### 出力例
```
発見したセグメント:
1. 「平日午前の高頻度購入者」（432人、平均LTV ¥18,500）
   → 平日朝の通勤時間にメール配信が効果的
2. 「夏季のみアクティブ」（128人、平均LTV ¥6,200）
   → 5-9月にキャンペーン集中
```

#### 実行頻度
週次（日曜深夜）

---

### 2-4. 異常検知エージェント（Anomaly Detection Agent）

#### 監視対象
- 開封率の急減（過去30日平均から -30% 以上）
- 配信失敗率の急増（5% 超）
- 購読解除の急増（日次平均の3倍以上）
- 配信遅延（cron が30分以上動いていない）
- バウンス率上昇

#### 通知先
Slack `#ferment-alerts` チャンネル

#### 反応速度
- 重要：5分以内
- 警告：15分以内

---

### 2-5. 学習エージェント（Learning Agent）

#### 役割
全配信結果を学習データに蓄積、次回判断を改善。

#### 学習データ
- subject_line_history（既存）
- send_time_history（時間帯×開封率）
- segment_performance（セグメント×開封率×売上）
- content_block_performance（テンプレ要素×CV率）

#### 適用先
- 戦略エージェントへのコンテキスト追加
- 文面生成エージェントへのプロンプト改善
- 配信時刻最適化アルゴリズム

## ⚖️ 4. ガバナンス設計（人間介入ポイント）

### 必ず人間承認が必要なもの
| アクション | 理由 |
|-----------|------|
| 1,000人以上への一斉配信 | 規模による影響大 |
| 新セグメントの初回配信 | 効果未知 |
| 削除・変更アクション | 不可逆 |
| 月予算超過の支出 | コスト制御 |
| ブランドメッセージの変更 | ブランド毀損リスク |

### AI 単独判断 OK のもの
| アクション | 条件 |
|-----------|------|
| 100人以下のセグメント配信 | 過去類似実績あり |
| 既存テンプレでの配信 | 件名 A/B のみ |
| 個別の Profile-Centric メール | 1対1 配信 |
| カゴ落ちリマインド | 既存フロー |
| 休眠復帰メール | 既存フロー |
| タグ付与・除去 | データ操作のみ |

## 🛡️ 5. リスク管理

### A. 暴走防止
- **配信レート制限**: 24時間で5,000通まで
- **AI 提案の連続実行制限**: 同一日 10アクション上限
- **Kill Switch**: ダッシュボードに「全自動停止」ボタン

### B. コスト制御
- **AI API 月予算**: ¥10,000 上限
- **超過時の挙動**: アラート → 自動停止
- **モニタリング**: 日次予算消化率を Slack 通知

### C. ブランド保護
- **NG ワードリスト**: AI 生成内容を必ずチェック
- **過剰送信防止**: 同一顧客に週3通以上は警告
- **送信前承認**: 1,000人超は必須

### D. 法令遵守
- **配信停止リンク**: AI 生成テンプレに自動挿入（既存）
- **GDPR 削除リクエスト**: 自動処理（既存）
- **特定電子メール法**: 送信者表示・連絡先明記をチェック
